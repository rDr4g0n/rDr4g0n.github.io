Active Dashboard demo

Created
March 2017

This active dashboard demo was designing to give users access to a tremendous amount of information, while providing tools to make it easy to focus on the important details.

Some key aspects of this UI include:

  • The "big graph" which permits exploration of time series data by streaming and resampling time series data from OpenTSDB as the user pans and zooms.
  • Cards which represent useful chunks of context. Cards focus on giving the user a "gut feel" of the data by using minimal visualizations such as sparklines (powered by QuickVis)
  • Mouse position is correlated across cards with a focus line

The UI for this demo was highly functional and feature rich, despite a tight deadline. Unfortunately the code for the dashboard and the streaming TSDB graphs are not available for review.

Phoxlights

Created
September 2015

Phoxlight is a customizable, full color LED bicycle brake light and turn signal. Software by myself and hardware by Julian Containne.

A ring of 16 RGB LED pixels serves as the taillight and displays right and left turn signal animations, brake light, and running animation. The running animation, colors, and patterns can be customized via web or mobile app.

The original Phoxlight, we fondly call "the medusa", was a tangled mess of wires connecting turn signal lights, switch, brake sensor, taillight and the Arduino microcontroller. Then along came the esp8266, a SOC with a much better processor, more RAM, and integrated WiFi. This presented the opportunity to WiFi all the things. Instead of running wires everywhere, each component connects to the taillight-hosted WiFi network.

The software architecture embraces this extendability, making it easy-peasy to design new components (think brake sensors, spoke lights, fork lights, or non-bike things like wifi-enabled jewelry, skate lighting, band costumes), and connect those components to one another. It's primarily written in C, but uses an Arduino C++ layer to abstract much of the hardware interaction. The animation library allows keyframed animations of bitmaps, with the ability to apply arbitrary "transformations" to the bitmap.

Features

  • 16 LED RGB light ring taillight
  • Turn signal switch component
  • Brake sensor component
  • Buy new components at any time and sync them with your taillight
  • Mobile app allows configuration of components
  • Phoxlight platform makes it easy to wire pieces together to create new components
  • Phoxlight animation library provides keyframed bitmap animation with arbitrary "transforms"
  • OTA upgradable firmware

Brownie Oven

Created
September 2014

Brownie Oven is a UI for authoring "brownies". Brownies, basically 3D sprites, were created by a coworker (Want webgl? You want Rye Terrell).

The idea with Brownie Oven was to make it very easy for middle school kids to draw in 3D. The approach I took to this problem is to let them "paint" in 2D slices, providing immediately visible results, while making it easier to reason about creating content in 3D space.

We finished a basic proof-of-concept and left behind a long list of features to implement one day (I'm still pretty interested in implementing a flood-fill algorithm, selections, and extrusion).

Features

  • Paint, Erase, Pan and Zoom around a 2D "slice" of the brownie
  • Live 3D preview with rotation controls and lighting
  • Import and export brownies
  • Simple scripting API to programatically draw brownies

FrameFighter

Created
September 2012

FrameFighter is a pair of hybrid mobile app, built with PhoneGap, that allows Ultimate Marvel vs Capcom 3 or Tekken Tag Tournament 2 players to compare moves and combos by how many frames they take to execute, and what state the player is left in after the move completes. Timing is critical in fighting games, and choosing a move that gives you a few extra frames advantage over your opponent can make or break a match.

Features

  • Complete moveset with commands, frame data, special effects, and more
  • A ton of custom icons
  • Combo builder
  • Compare moves to see which has the advantage

SFNinja

Created
June 2011

Marking my first entry into application user interface design was SFNinja. Our team of designers was stuck using Salesforce's very generic and very frustrating website to fill Creative Requests from the sales team. After discovering that SalesForce offered a Flash/Flex SDK for interacting with their database, I evaluated our workflow and designed a UI that gets out of the way and lets work get done.

The resulting UI was precisely aimed at making it faster and easier for designers to communicate with sales reps and get work published.

Features

  • Sort, filter, and customize list of Creative Requests
  • Saved searches
  • Tabbed management of Creative Requests
  • Carefully designed Creative Request view, focused on pushing important stuff up front
  • Common workflows baked into the UI (eg: if you're attaching files, I assume you want to change the status to "Awaiting Approval")
  • Bulk upload and download attachments (a feature that was absent from Salesforce at the time)
